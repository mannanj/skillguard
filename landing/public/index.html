What it catches

Found in the wild

When Koi Security audited ClawHub, it found 341 malicious skills — typosquats shipping infostealers. Snyk's ToxicSkills study of the same registry found a critical issue in 13.4% of all skills. Claude Code skills share the same trust model: files you probably never read, running with your permissions.

  • Prompt injection in SKILL.md
  • Credential & env-var exfiltration
  • Reverse shells & hidden subprocesses
  • Zero-width & base64 obfuscation
  • Persistence & container escape
  • curl | sh pipe execution

How it works

A scanner, a gate, and rules you can read

Scans every file in every skill

A zero-dependency local engine checks 60+ readable patterns across 12 threat categories. Optional Cisco, Snyk, and SkillAudit engines add second opinions when you want them.

Blocks via PreToolUse hook

Unscanned skills are refused before they run, and a skill modified after its last scan is blocked until you re-scan. Skipping is possible — but it's an explicit choice, never the default.

Rules you can read

Local by default — file contents leave your machine only if you enable a cloud engine. Every local detection is a pattern you can open and audit, and false positives are one --mark-fp away.
